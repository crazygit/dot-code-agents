# Go 测试与校验

执行 Go 验证: $ARGUMENTS

## 工作流程

1. 识别当前仓库是否为 Go 模块，优先查看 `go.mod` 和相关包路径。
2. 如果提供了目录或包路径，先只验证该范围，再决定是否扩展到 `./...`。
3. 默认先运行测试，再补充 `go vet` 或必要的静态检查。
4. 如果测试失败，先定位到具体包和失败原因，再继续扩大排查。

## 默认行为

- 无参数时，优先对整个模块执行 `go test ./...`
- 如果变更只影响局部包，优先对相关包执行更小范围测试
- 如果当前目录不是 Go 模块，先说明原因，不要盲目执行命令

## 关注点

- 单元测试是否通过
- 编译是否正常
- 是否存在明显的 vet 问题
- 是否需要补充针对变更的最小测试用例
