# 代码审查 (Code Review)

审查当前变更或指定范围: $ARGUMENTS

## 工作流程

1. 先读取 `git status`、`git diff` 和最近提交，确认本次变更范围。
2. 只聚焦修改过的文件，优先检查安全性、正确性、可维护性和性能。
3. 如果提供了 PR 编号或提交范围，优先围绕该范围展开审查。
4. 按 `Critical`、`Warning`、`Suggestion` 输出结果。
5. 每条反馈都给出具体文件和可执行的修复建议。

## 审查重点

- 安全性：认证、授权、注入、敏感信息泄露
- 正确性：边界条件、错误处理、并发、资源释放
- 可维护性：命名、重复代码、复杂度、注释质量
- 性能：明显的重复计算、无效 IO、低效循环

## 输出要求

- 先给结论，再给细节
- 只审查真实问题，不要为了凑条目而输出低价值建议
- 如果没有发现问题，明确说明当前未发现高风险项，并列出残余风险或测试空白
